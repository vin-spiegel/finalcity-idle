# 🚀 Spiegel의 작업 체크리스트

코딩 작업을 시작하기 전에 꼭 확인해야 할 목록입니다. 친구가 알려준 소중한 팁들을 여기에 기록합니다!

---

### ⚡️ 퀵 체크리스트 (바쁠 땐 이것만 보세요!)
1.  **📥 최신화:** `git pull origin main` (다른 사람이 고친 거 가져오기)
2.  **🌿 실험실 만들기:** `git checkout -b 내-작업-이름` (나만의 작업 공간 만들기)
3.  **🛠️ 마음껏 고치기:** 코드나 파일 내용 수정하기
4.  **📤 검사 맡기:** `git push origin 내-작업-이름` (서버에 올리고 풀리퀘 보내기)

---

### 1. 📥 작업 시작 전 `git pull` 하기
*   **무슨 뜻인가요?** 
    다른 사람이 고쳐놓은 최신 내용을 내 컴퓨터로 가져오는 작업이에요. 
*   **왜 해야 하나요?** 
    이걸 안 하면 나중에 코드가 꼬여서(충돌) 아주 복잡한 상황이 생길 수 있어요. 항상 작업을 시작하기 전에 "메인 브랜치"에서 이걸 먼저 하세요!
*   **어떻게 하나요?** 
    ```bash
    git pull origin main
    ```

---

### 2. 🌿 나만의 작업 공간(브랜치) 만들기
*   **무슨 뜻인가요?**
    '메인 도로'에서 잠시 옆길로 빠져나와 나만의 **'비밀 실험실'**을 만드는 거예요. 
*   **왜 해야 하나요?**
    메인 코드를 직접 건드리면 망가졌을 때 복구하기 힘들지만, 내 브랜치에서 작업하면 마음껏 실수해도 괜찮아요!
*   **어떻게 하나요?** (터미널에 입력)
    ```bash
    git checkout -b 내-작업-이름
    ```
    *(예: `git checkout -b fix-ui-bug` 처럼 작업 내용을 이름으로 쓰면 좋아요!)*

---

### 3. 🛠️ 마음대로 코드 바꾸기
*   **무슨 뜻인가요?**
    이제 내 실험실이 생겼으니, 파일들을 열어서 내용을 고치고 싶은 대로 고치면 돼요. 
*   **팁:**
    중간중간 "이만큼 고쳤어!"라고 저장(Commit)하는 걸 잊지 마세요.

---

### 4. 📤 검사 맡기(Pull Request 올리기)
*   **무슨 뜻인가요?**
    작업이 다 끝났으면, "제가 이렇게 고쳤는데 메인 코드에 합쳐도 될까요?"라고 친구들에게 물어보는 과정이에요. 이걸 줄여서 **'풀리퀘(PR)'**라고 불러요.
*   **어떻게 하나요?**
    1. 고친 내용을 서버에 올립니다 (`git push origin 내-작업-이름`).
    2. 깃허브(GitHub) 웹사이트에 들어가면 나오는 초록색 버튼(**Compare & pull request**)을 누르면 끝!

---

### 🚨 접속/로그인 트러블슈팅 (문제가 생기면 보세요!)

화면이 안 뜨거나 로그인이 안 될 때는 당황하지 말고 아래 순서대로 해보세요.

#### 1. 해결 방법 (5단계)
1.  **중단:** 지금 돌아가는 터미널 창에서 **Ctrl+C** (또는 Esc)를 눌러서 멈춥니다.
2.  **AI 요청:** AI에게 **"서버 포트 전부 종료해줘"**라고 말하세요.
3.  **새 창 열기:** 새 터미널 창을 하나 더 엽니다.
4.  **실행:** 새 창에서 **`pnpm dev`**를 입력하고 엔터를 칩니다.
5.  **확인:** 브라우저에서 다시 접속해 봅니다.

#### 2. 왜 이런 문제가 생기나요?
> "AI는 보안상 구글 로그인 권한이 없어서 직접 서버를 실행할 수 없어요. 사용자가 직접 실행해야 정상적으로 작동합니다!"
