# 파이널시티 — PRD v2.0 스킬 시스템 설계

> 참조: 일랜시아(Ilaria) 잡포인트 시스템
> prd1.md의 레벨 시스템을 대체

---

## 1. 핵심 철학

**"레벨이 없다. 무엇을 했느냐가 캐릭터다."**

일반 RPG의 레벨(종합 성장 지표)을 제거하고,
**행동 = 스킬 성장 = 구역 해금**으로 직결되는 구조.

플레이어는 레벨을 올리는 게 아니라,
특정 장소에서 특정 행동을 반복해 그 행동에 대한 숙련도를 쌓는다.

---

## 2. 스킬 시스템

### 2.1 스킬 레벨

- 범위: `0.00 ~ 100.00`
- 소수점 둘째 자리까지 표시
- 1 틱 = 1 잡포인트(jobPoint), 100포인트 = 1.00 숙련도
- 표시 형식: `탐험  3.45` (Lv. 접두사 없음, 숫자만)

```
구역 탐색 → 틱 발생 → jobPoints +1 → skillLevel = jobPoints / 100
```

### 2.2 현재 스킬 종류

| jobType | actionType | 설명 |
|---------|------------|------|
| searcher | 탐험 | 폐건물·위험 구역 수색 |
| technician | 작업 | 기계 조작·장비 제작 |
| scholar | 조사 | 마나 분석·유물 연구 |
| *(예정)* lumberjack | 벌목 | 산림 자원 채취 |
| *(예정)* miner | 채굴 | 광물·마나석 채굴 |
| *(예정)* fisher | 낚시 | 특수 자원 획득 |

### 2.3 스킬 성장 규칙

- 한 구역은 하나의 스킬만 훈련 (구역당 1 액션)
- 스킬은 해당 스킬이 연결된 구역에서만 오름
- 오프라인 중에도 틱이 쌓이므로 방치 효율 = 숙련도 상승
- 현재 스킬 상한: 100.00 (이후 마스터리 시스템으로 확장 예정)

## 3. 구역 구조와 스킬 잠금

### 3.1 구역-스킬-잠금 연결

```
구역(zone)
  └── actionType  : 액션 버튼 라벨 (탐험, 작업, 조사 ...)
  └── jobType     : 어떤 스킬이 오르는가 (searcher, technician ...)
  └── levelReq    : 진입 요건 — 해당 jobType 스킬이 X.XX 이상이어야 함
                    (0 = 누구나 진입 가능)
```

### 3.2 현재 구역 & 잠금 설계

| 구역 | 액션 | 스킬 | 진입 조건 | 틱(s) | 위험도 |
|------|------|------|-----------|--------|--------|
| 상업 구획 폐건물 | 탐험 | searcher | 없음 | 8 | 안전 |
| 구 제조 공장 지하 | 작업 | technician | 없음 | 18 | 보통 |
| 마나 균열 지대 | 조사 | scholar | 없음 | 25 | 위험 |
| 고대 연구소 잔해 | 조사 | scholar | scholar 15.00 | 35 | 위험 |
| 공허 구역 심층부 | 탐험 | searcher | searcher 20.00 | 50 | 극한 |

**진행 경로 예시:**
```
탐험 시작(상업 구획, 8s) → searcher 0.00→20.00 달성 → 공허 심층부(50s) 해금
조사 시작(마나 균열, 25s) → scholar 0.00→15.00 달성 → 고대 연구소(35s) 해금
```

### 3.3 브랜치 노드(지역)의 잠금

- 현재: 브랜치(키르타스 평원, 붉은 협곡 등)는 항상 탐색 가능
- 추후: 브랜치 자체도 특정 스킬 요건 추가 가능
  (예: 붉은 협곡 진입에 searcher 5.00 이상 요구)

## 4. 스킬 → 스탯 변환 (잡포 시스템)

prd1에서 정의한 스탯 변환 로직을 스킬 레벨 기준으로 재기술.

> `스킬 레벨 X.00` = `잡포인트 X×100`

### 4.1 수색자 (searcher)

| 스킬 레벨 | 효과 |
|-----------|------|
| +3.00마다 | 탐지력 +1 (최대 +30) |
| +10.00마다 | 체력 +1 (최대 +10) |
| +5.00마다 | 마나감응 -1 (최대 -10) |
| 20.00 달성 | 스킬 해금: 은닉물 감지 |
| 75.00 달성 | 스킬 해금: 위험 예측 |
| 100.00 달성 | 마스터 (이후 확장) |

### 4.2 학자 (scholar)

| 스킬 레벨 | 효과 |
|-----------|------|
| +3.00마다 | 마나감응 +1 (최대 +30) |
| +5.00마다 | 탐지력 +1 (최대 +10) |
| +5.00마다 | 체력 -1 (최대 -10) |
| 20.00 달성 | 스킬 해금: 유물 감정 |
| 75.00 달성 | 스킬 해금: 마나 정제 |

### 4.3 기술자 (technician)

| 스킬 레벨 | 효과 |
|-----------|------|
| +3.00마다 | 제작력 +1 (최대 +30) |
| +10.00마다 | 체력 +1 (최대 +10) |
| +5.00마다 | 마나감응 -1 (최대 -10) |
| 20.00 달성 | 스킬 해금: 기계 해체 |
| 75.00 달성 | 스킬 해금: 장비 강화 |

> **스탯 시스템은 아직 미구현.** DB에 `user_stats` 테이블은 존재하나 연산 로직 없음.

## 5. UI 설계

### 5.1 리프 뷰 (구역 진입 화면)

```
┌─────────────────────────────────┐
│ 탐험   3.45          (필요 20.00)│  ← nav-skill-row: 스킬명 + 현재 숙련도
├─────────────────────────────────┤
│ ▶ 탐험 시작                  ▶ │  ← 액션 버튼 (actionType 기반)
│   ◷ 8s  ■ 안전                  │
├─────────────────────────────────┤
│ ✕ 탐험 취소                  › │  ← 탐색 중일 때만 표시
│   ◷ 00:02:15  12.3%             │
└─────────────────────────────────┘
```

- 스킬 레벨 숫자 색: `var(--cyan-dim)`
- 잠금 조건 색: `var(--amber)`
- 잠금 상태: `🔒 탐험 20.00 필요` + 버튼 opacity 0.45

### 5.2 브랜치 리스트 (구역 목록)

```
상업 구획 폐건물     탐험  ◷ 8s  ■ 안전       ▸
구 제조 공장 지하    작업  ◷ 18s ■ 보통        ▸
마나 균열 지대       조사  ◷ 25s ■ 위험        ▸
고대 연구소 잔해     조사  ◷ 35s ■ 위험   🔒   ▸
공허 구역 심층부     탐험  ◷ 50s ■ 극한   🔒   ▸
```

- `Lv.N` 배지 없음 — 숫자 레벨 표시 지양
- 잠금 아이콘으로만 진입 불가 암시, 구체 조건은 진입 후 표시

### 5.3 스킬 탭 (추후 구현)

프로필 탭 내 별도 섹션 또는 독립 탭:

```
스킬                   숙련도       진행도
─────────────────────────────────────────
탐험 (수색자)          3.45         ████░░░░░░░░  3.45/100
작업 (기술자)          0.00         ░░░░░░░░░░░░  0/100
조사 (학자)            0.00         ░░░░░░░░░░░░  0/100
```

## 6. 데이터 모델 (현재 구현)

```sql
-- zones
action_type  TEXT          -- 액션 버튼 라벨 (탐험/작업/조사)
job_type     TEXT          -- 스킬 종류 (searcher/technician/scholar)
level_req    INTEGER       -- 진입 요건 (jobType 스킬 레벨, 0=제한없음)
tick_sec     INTEGER       -- 틱 간격(초)

-- user_jobs
job_type     TEXT          -- 스킬 종류
job_points   INTEGER       -- 누적 포인트 (÷100 = 스킬 레벨)
is_active    BOOLEAN       -- 현재 활성 스킬 여부
is_mastered  BOOLEAN       -- 100.00 달성 여부 (미사용)

-- skillLevel = job_points / 100
```

## 7. 미구현 / 다음 단계

### Phase A — 스킬 시스템 완성
- [ ] 스킬 → 스탯 자동 변환 (`userStats` 업데이트 로직)
- [ ] 스킬 탭 UI (전체 스킬 목록 + 진행도)
- [ ] 스킬 해금 포인트 도달 시 로그 표시 (`20.00 달성 — 은닉물 감지 해금`)
- [ ] 스킬 레벨로 브랜치 노드 잠금 (선택)

### Phase B — 구역 확장
- [ ] 벌목 구역 (lumberjack 스킬)
- [ ] 채굴 구역 (miner 스킬)
- [ ] 구역별 드롭 테이블 다양화

### Phase C — 직업 전환 시스템
- [ ] 스킬 전환 시 포인트 50% 유지 (일랜시아식)
- [ ] 전환 비용 (BSS)
- [ ] 전환 쿨다운
- [ ] 마스터 달성 시 스탯 보너스 영구 적용

### Phase D — 복합 스킬 (2차 직업)
- [ ] 2개 스킬 마스터 → 복합 스킬 해금
  - 수색자(100) + 학자(100) → 마나 공학자
  - 수색자(100) + 기술자(100) → 유물 복구사
- [ ] 복합 스킬 전용 구역 추가

## 8. 밸런스 레퍼런스

| 스킬 레벨 | 필요 틱 | 8s 틱 기준 시간 | 25s 틱 기준 시간 |
|-----------|---------|-----------------|-----------------|
| 1.00 | 100 | 13분 | 42분 |
| 5.00 | 500 | 67분 | 3.5시간 |
| 15.00 | 1,500 | 3.3시간 | 10.4시간 |
| 20.00 | 2,000 | 4.4시간 | 13.9시간 |
| 50.00 | 5,000 | 11.1시간 | 34.7시간 |
| 100.00 | 10,000 | 22.2시간 | 69.4시간 |

> 방치형 특성상 실제 체감 시간은 오프라인 시간 포함.
> 8s 틱 구역(상업 구획)이 스킬 레벨 올리기 가장 빠름 → 진입 구역으로 적합.

## 9. 일랜시아 참조 메모

- 스킬이 올라가는 유일한 방법 = 해당 행동 반복 (Use-based progression)
- 레벨업 개념 없음, 모든 성장은 개별 스킬
- 직업 = 어떤 스킬에 집중하는지의 선택, 언제든 전환 가능
- 스킬 15.00+ 달성 시 직업(job) 인정 → 직업 포인트 누적 시작 (우리 버전과 동일)
- 일랜시아는 소수점 2자리 숙련도 표기 (우리 그대로 채택)
- 구역마다 1개 메인 액션, 스킬이 엮이는 구조 (우리 actionType/jobType 방식 동일)
